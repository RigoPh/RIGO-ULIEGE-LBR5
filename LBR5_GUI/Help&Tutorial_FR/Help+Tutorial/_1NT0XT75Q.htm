Optimisation

Cet exemple permet de guider l'utilisateur lors de la création d'un modèle sur l'interface de LBR5.
Tout d'abord il faut construire la géométrie. Pour ce faire, suivre l'élaboration de la géométrie dans l'exemple.
Nous allons ensuite entrer les caractéristiques liées à l'optimisation du modèle.

Dans cet exemple, la fonction objectif est le coût. On y trouve des restrictions structurelles, géométriques et d'égalité.

 Généralités de l'optimisation :

Cliquer dans "project" sur "options" puis "optimization", la fenêtre suivante apparaît :

             

  • Pour optimiser l'échantillonnage du modèle, cocher "perform optimization".
  • Choisir le nombre d'itérations désirées pour le calcul.  Nous choisissons 5 itérations.
  • Choisir le type de fonction objectif. Nous choisissons la fonction objectif poids minimum simplifié (Minimal Cost Simplified).

    • Données et variables de l'optimisation :

    Si la fonction à minimiser est le coût, certaines données relatives aux coûts sont à introduire. Néanmoins, les valeurs affichées par défaut sont tout à fait valable.

  •  Cliquer sur "optimization" puis "cost" puis "Simplified" dans le cas d'une fonction objectif de coût simplifié (sinon utiliser "Developped") 

  • Entrer les différentes valeurs de coût.

    •  

     

  • Cliquer ensuite sur "optimization", puis "design variables" pour déterminer les variables de dimensionnement propres à chaque panneau.

    •      

    Remplir les limites inférieures et supérieures des variables de dimensionnement choisies pour le panneau. Dans cet exemple, il y a 9 variables de dimensionnement par panneau.

    Pour faciliter l'écriture des variables, choisir les variables à copier en les sélectionnant (elles deviennent soulignées), click droit sur la souris et "copier la sélection" puis choisir le ou les panneaux qui ont les mêmes limites de variable et cliquer droit sur la souris et "paste la sélection".

    Cliquer sur "apply" pour enregistrer les limites des variables.

    Restrictions :

  • Restrictions Structurelles : Pour définir les restrictions structurelles à appliquer, cliquer sur "optimization", puis "structural constraints".

    • Les restrictions structurelles sont a entrer pour chaque cas de charge. Le point non extrème où peuvent être appliquées les restrictions sont à définir pour chaque panneau.

    Pour ajouter une restriction, cliquer sur "add restriction". Pour oter une restriction sélectionnée , cliquer sur "remove restriction". Pour accélérer l'entrée des restrictions, cliquer droit sur la case "restrictions" puis cliquer sur "copy" ou "copy all restrictions to other panels" (coller ou coller toutes les restrictions aux autres panneaux). Sélectionner les panneaux qui comportent les mêmes restrictions que le panneau de référence et coller (click droit "paste"). De même cliquer droit sur le cas de charge dont les restrictions sont déjà définies et cliquer sur "copy paste to all load cases" pour que toutes les restrictions soient appliquées également dans tous les cas de charge.

    Le logiciel LBR5 ne comporte pas de restrictions liées aux épontilles. Dans notre exemple, les panneaux 18, 19 et 20 sont des épontilles et ne possèdent donc pas de restrictions.

    Pour connaitre le numéro de référence de la restriction à imposer, cliquer sur "references".

    Cliquer sur "apply" pour enregistrer toutes les données.

  • Restrictions Géométriques : Pour définir les restrictions géométriques à appliquer, cliquer sur "optimization", puis "geometrical constraints".

    •       

    Définir les restrictions en cochant les cases adéquates. Pour obtenir une phrase explicative des différentes restrictions possibles, cliquer sur "references". Les différentes variables utilisées sont illustrées dans "show figure".

    Pour accélérer l'édition des restrictions, cliquer droit dans la case des restrictions et cliquer sur "copy", puis choisir les panneaux qui recevront les mêmes restrictions et cliquer droit sur "paste".

    Dans notre cas nous utiliserons un set de restriction (Hughes).

       

    Cliquer sur le panneau qui comporte des restrictions, cocher les restrictions , cliquer sur "apply" puis remplir ISEMA et ISEMR.

    Cliquer sur "apply" pour enregistrer les données.

  • Restrictions d'égalité : Pour définir les restrictions d'égalité à appliquer, cliquer sur "optimization", puis "equality constraints".

    •    

     Sélectionner le panneau dépendant et le panneau indépendant, cliquer sur les variables communes semblables (elles se soulignent), choisir la proportion entre les panneaux, cliquer sur "=".

    Cliquer sur "apply" pour prendre en compte les données, le nombre total de restrictions entrées est indiqué, une liste énumère les restrictions souhaitées.

    Dans notre cas le ratio est toujours égal à 1. Il y a 69 restrictions au total.

        

     

  • Restrictions sur la position du centre de gravité : Pour définir les restrictions sur le centre de gravité, cliquer sur "optimization", puis "Gravity center".

    • Dans cet exemple, il n'y a pas de restrictions concernant le centre de garvité.

           

     

    Résolution : 

  • Avant de résoudre le modèle donné, nous allons le sauver. Cliquer sur "file" puis "save as" et sauver le fichier dans le répertoire choisi. Ce fichier a une extension .lbr. Dans le coin supérieur gauche de l'interface LBR5, le fichier ne s'appelle plus "projet1.lbr" mais bien le nom donné lors du sauvetage.
  • Certaines options sont vérifiées par "tools", "check model". 
  • Cliquer sur "Analysis" puis "solve" pour lancer la résolution. Le message suivant apparaît, il permet de vérifier que le modèle a bien été sauvé. cliquer sur "oui".

    •                 

    LBR5 résoud le modèle. Il n'est pas possible d'avoir accès à l'interface pendant que LBR5 tourne. Lorsque la fenêtre DOS a disparu, les résultats sont accessibles.

    Résultats :

    Les résultats sont disponibles sous forme de graphe ou de fichier texte. Cliquer sur "Analysis" puis respectivement sur "output graphs" ou "output files".

    Les fichiers sol-, sol2- et opt- sont automatiquement enregistrés sous format texte dans le dossier relatif au fichier de données.
