Géométrie

Cet exemple permet de guider l'utilisateur lors de la création d'un modèle sur l'interface de LBR5.
Nous allons tout d'abord construire la géométrie du modèle. Nous pourrons ainsi observer les résultats en fonction de l'échantillonage que nous avons choisi.

 Avant-Propos :

  • Ouvrir l'interface LBR5, on obtient une fenêtre dont la barre des menus est la suivante:

    •                      

  • Cliquer sur "File" et choisir si on utilise un nouveau modèle ou un modèle déjà existant.
  • Nous avons ainsi la barre de menu suivante :

  •  Cliquer sur "project" puis sur title pour introduire le nom du modèle. introduire le nom dans l'espace prévu puis cliquer sur "OK".

    •                             

     

    Généralités de la structure:

  • Cliquer sur "project", "options" puis "optimization" pour vérifier que l'optimisation de la structure ne sera pas effectuée. Pour avoir une simple analyse, décocher "perform optimisation".

    •                           

  • Cliquer sur "project", "options" puis sur "DeadWeight". Cocher "Consider Deadweight" pour que le poids propre de la structure intervienne dans le calcul.

    •                          

  • Cliquer sur "project", "options" puis sur "Fourier Series" pour déterminer le nombre de termes de fourier désirés lors du calcul. Nous choississons "-7" dû à la symétrie de la coupe.

    •                           Géométrie

     

    Création de la géométrie :

  • Cliquer sur "project", "options" puis sur "overall span" pour déterminer la longueur totale (selon OX) et les sections de calcul de la structure. Puis "OK" pour enregistrer les données.

    •                      

     

  • Cliquer sur "Model" puis sur "nodes" pour entrer les coordonnées des noeuds.

    •                         

    On obtient alors le modèle fait uniquement de noeuds. (si les noeuds ne sont pas visibles, cliquer sur   ou vérifier leur impression dans "Display Options".)

  • Cliquer sur "Model", puis "add plate(s)" pour ajouter une plaque entre deux noeuds (add beam pour ajouter une épontille, add double hull pour ajouter une double coque). Dans la barre des messages, il est indiqué "Function mode = add plate: First Node", cliquer sur le noeud origine du panneau. Dans la barre des messages, il est maintenant indiqué "Function mode = add plate: Second Node", cliquer sur le noeud arrivée du panneau.

    • Lorsque tous les panneaux sont définis, cliquer sur escape si nécessaire pour sortir du mode "add plate".

  •  Pour entrer la géométrie détaillée relative à chaque panneau, cliquer sur "panels" puis "plate scantlings" dans le cas d'une plaque ("beam scantling" pour une épontille et "double hull scantling" pour une double coque ). Le tableau suivant apparaît:

    •     

    La géométrie ( geometry) est donnée par défaut en fonction des coordonnées des noeuds.

  • Caractéristiques des plaques: Entrer les épaisseurs des tôles (thickness), les dimensions et positions des raidisseurs (stiffeners set 1), des cadres (frames set 1), des raidisseurs et cadres secondaires, des traverses (girders) , les caractéristiques des matériaux utilisés (materials) ainsi que le coefficient de participation à la flexion globale (participation). Dans cet exemple nous n'avons ni raidisseurs et cadres secondaires, ni traverses.

    •  

  • Caractéristiques des épontilles : Entrer le type de section de l'épontille (sections) et les caractéristiques des matériaux utilisés (materials).

    • remarque 1: nous n'avons pas de doubles coques dans cet exemple.

    remarque 2: pour accélérer l'entrée des données les caractéristiques d'un panneau peut-être copiée puis collée sur différents panneaux.

    On obtient le modèle suivant :

     

  • Pour introduire les conditions d'appui, cliquer sur "boundary conditions" dans "panels". Le tableau suivant apparaît :

    •                                    

    Seuls les panneaux ayant une extrémité non liée à un autre panneau peuvent avoir des conditions d'appui. Double cliquer sur "free edge" pour obtenir la liste des conditions d'appui possibles.

    Dans le cas de cet exemple, les conditions de bord des 5 panneaux sont choisies égales à des conditions de symétrie. (Symmetry axis 1).

    Cas de charge :

    Nous avons 2 cas de charges à envisager. Cliquer sur "Loads" et "load cases". Cliquer 2 fois sur "add", donner un nom aux différents cas de charge et vérifier s'ils sont pris en compte ou pas dans le calcul (state).

                        

     Ensuite cliquer sur "loads" dans "loads" pour détailler le chargement. Déterminer les pressions uniformément distribuées, les pressions distribuées par pas, les pressions localisées et les moments d'extrémité.

    Dans le cas de cet exemple, les pressions uniformément réparties sont les mêmes dans les 2 cas de charge. Il n'y a pas de pressions distribuées par pas ni de pressions localisées. Le cas de chargement 1 comprend des moments d'extrémité en hogging, le cas de charge 2 en sagging.

             

    Niveau de détails des résultats :

    Pour choisir le niveau de détails des résultats, cliquer sur "project", puis "options" et enfin "output". Dans cet exemple nous choisissons un niveau 4 de résultats.

                                

     Résolution :

  • Avant de résoudre le modèle donné, nous allons le sauver. Cliquer sur "file" puis "save as" et sauver le fichier dans le répertoire choisi. Ce fichier a une extension .lbr. Dans le coin supérieur gauche de l'interface LBR5, le fichier ne s'appelle plus "projet1.lbr" mais bien le nom donné lors du sauvetage.
  • Certaines options sont vérifiées par "tools", "check model". 
  • Cliquer sur "Analysis" puis "solve" pour lancer la résolution. Le message suivant apparaît, il permet de vérifier que le modèle a bien été sauvé. cliquer sur "oui".

    •                 

    LBR5 résoud le modèle. Il n'est pas possible d'avoir accès à l'interface pendant que LBR5 tourne. Lorsque la fenêtre DOS a disparu, les résultats sont accessibles.

    Résultats :

    Les résultats sont disponibles sous forme de graphe ou de fichier texte. Cliquer sur "Analysis" puis respectivement sur "output graphs" ou "output files".

    Les fichiers sol- et sol2- sont automatiquement enregistrés sous format texte dans le dossier relatif au fichier de données.
